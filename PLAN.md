## 1. 專案概述

一個「後臺規劃、前臺展示」的自助旅遊記錄網站：

- **後臺（私人）**：行前規劃（航班、住宿、接送、準備清單）＋旅途中/旅途後上傳照片、AI 協助生成遊記草稿
- **前臺（半公開）**：以「旅遊文」形式展示每趟行程——行前情報、航班轉機簡介、住宿與景點介紹、交通經驗、照片牆
- **延伸產出**：每趟行程可匯出 A5 可印刷 PDF 旅遊小書

## 2. 技術架構（已定案）

| 元件 | 技術 | 備註 |
|------|------|------|
| 前臺網站 | GitHub Pages 靜態網站 | 加 `<meta name="robots" content="noindex">` 做半公開（不進搜尋引擎、有連結可看） |
| 後臺 | 同站隱藏管理頁 | Supabase Auth 登入（沿用奶貓日記模式） |
| 資料庫 | Supabase Database + RLS | 公開欄位/私人欄位分離（見 §4 隱私） |
| 照片儲存 | Supabase Storage | 前端壓縮後上傳（見 §6） |
| AI 服務 | Gemini API（使用者已有付費金鑰） | ⚠️ 金鑰只存於 Supabase Edge Function 環境變數，絕不進前端程式碼 |
| 地圖/景點 | Google Places API | 住宿搜尋帶入、周邊景點建議（免費額度內） |

## 3. 資料模型（Phase 1 依此建表）

一切以 **Trip（旅程）** 為單位。

- `trips`：標題、目的地、日期範圍、狀態（`planning` / `traveling` / `done`）、封面照、**theme（JSONB：主色/強調色/紙張底色 CSS 變數值）**、是否公開
- `flights`：所屬 trip、航段、航空公司（下拉＋可手填）、航班號、起降時間、轉機資訊、**轉乘方式**（直飛/轉機行李直掛/轉機需提領重掛）、**票種**（一般/四腿票/境外票/廉航/里程票/其他手填）；**私人欄位**：訂位代號等
- `stays`：所屬 trip、名稱、Google Place ID、地址座標、入退房日期；**私人欄位**：訂單編號
- `transport_cards`：所屬 trip、類型（接送/租機車/火車/渡輪…）、標題、內容、費用參考 — 前臺以醒目資訊框呈現（對讀者最有價值的內容，獨立成類）
- `photos`：所屬 trip、日期、**來源網址欄位**（預留日後搬遷至 Cloudflare R2 等，搬家只改網址不改程式）、說明
- `posts`：所屬 trip、類型（行前情報/每日遊記/總結）、狀態（草稿/已發布）、內容、AI 草稿原文
- `expenses`：所屬 trip、類別（住宿/機票/餐飲/交通/門票/購物/其他）、品名、備註、原幣金額＋幣別、臺幣約當（選填手動）、消費日期、可選擇性關聯到某筆住宿/航班/交通卡片（被關聯項刪除時保留記帳）；**整表僅本人可讀寫，前臺永不撈取**

## 4. 隱私規則（強制）

- 前臺 = 半公開（noindex + 有連結可看）；**不做密碼鎖**，以維持 FB 分享時預覽圖可正常抓取
- 訂位代號、訂單編號、任何個資只存私人欄位，RLS 限本人讀取，前臺查詢永不撈取
- 記帳（`expenses`）整表視同個資：RLS 僅本人，即使旅程公開也不顯示；日後若要公開花費統計，另開專用公開欄位
- Gemini 付費金鑰：只放 Edge Function 環境變數，Edge Function 驗證登入後才可觸發 AI 功能

## 5. 已定案的功能決策

| 功能 | 決策 |
|------|------|
| 航班異動更新 | **半自動**：① 後臺「查詢航班狀態」按鈕手動查詢 ② 出發前以 Claude Gmail 連接器掃描航空公司異動通知信，人工確認後更新。不使用付費即時航班 API |
| FB 粉絲頁發文 | **B 方案（半自動）**：後臺「產生 FB 貼文」按鈕 → AI 依粉專敘事風格生成貼文文字＋前臺連結 → 使用者複製貼上發文。Graph API 全自動列 Phase 5 選配 |
| AI 遊記 | 照片＋使用者口述重點 → Gemini 生成**草稿** → **必經人工確認/潤飾才發布**（不允許 AI 直接發布） |
| 歷史行程回填 | 後臺提供「匯入歷史行程」模式：一次貼上航班/住宿/照片/交通備註，AI 整理成遊記草稿 |
| 內容狀態 | 規劃中的 Trip 前臺顯示「行前情報文」；完成後同一 Trip 累積為完整遊記（無縫轉換） |

## 6. 照片策略

- 前端上傳時自動壓縮：長邊 1600px、約 250–350KB/張
- 500 張/趟 ≈ 150MB → Supabase 免費 1GB 約可容納 5–6 趟
- **原始檔不上傳**，留在使用者 Google 相簿/硬碟；印刷小書時才使用原始檔
- 容量將滿時遷移舊行程照片至 Cloudflare R2（免費 10GB），靠 `photos.來源網址` 欄位無痛搬遷

## 7. 風格憲章（前臺與小書共用；Phase 2 前臺開發依據）

- **核心原則：像日記、不像設計稿** — 照片是主角（版面佔比 ≥70%），不裁切破壞真實感；手繪元素是註記層不是主體
- **質感基底**：牛皮紙/米白紙張底色、格紋或點陣暗紋、紙膠帶、撕邊便條、白色描邊貼紙感照片框
- **手寫層**：景點/美食卡可加短評標註（箭頭＋一句話），字體用手寫感中文字型，文字要少、留白要多
- **插畫點綴**：低飽和暖色水彩風插圖，僅用於旅程封面、章節扉頁、小結區塊，每趟 3–5 張為上限（Phase 3 由 Gemini 依該趟主題色生成）
- **主題色：沿用既有 per-trip theme 系統** — 色調存於各 Trip 的 `theme` 欄位，以 CSS 變數（`--main` / `--accent` / `--paper`）套用；紙張底色一律維持淺色米白系（確保字體對比與印刷品質，避免深色底吃墨），僅主色與強調色隨旅程變化
  - 後臺色調編輯三層：① 預設調色盤（海島/山林/城市/雪國…）② 封面照片自動取色 ③ 色票手動微調
- 照片建議比例：3:2 或 4:3 為主，直式照片保留給滿版頁
- 開發前臺 UI 時使用 **frontend-design Skill**

## 8. 旅遊小書規格

- 開本：**A5 直式（14.8 × 21 cm）**
- 匯出 PDF 含 **3mm 出血**，可直接自印或送印刷廠
- 視覺語言與前臺共用（同一套 theme CSS 變數）
- 排入 Phase 4

## 9. 分階段開發計畫

| 階段 | 內容 | 產出/里程碑 |
|------|------|------------|
| Phase 1 | Supabase 建表（§3）＋ RLS ＋ 後臺登入 ＋ 手動輸入航班/住宿/交通卡片 ＋ 照片上傳（含前端壓縮） | 馬爾地夫＋普吉島行前資料先建入 |
| Phase 2 | 半公開前臺（noindex）＋ Trip 展示頁（手帳感版型＋主題系統）＋ Google Places 住宿/景點抓取 | 前臺上線 |
| Phase 3 | Edge Function 串 Gemini（金鑰安全）＋ AI 遊記草稿 ＋ FB 貼文產生器 ＋ 歷史行程回填 | 上次行程補入 |
| Phase 4 | 航班狀態查詢按鈕 ＋ Gmail 異動信解析流程 ＋ A5 PDF 小書匯出（含出血） | 可印刷成品 |
| Phase 5（選配） | FB Graph API 全自動發文 | — |
